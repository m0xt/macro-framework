## 2026-05-16 — Snapshot retention policy
Reason: daily snapshots are durable history for dashboard review and Supabase backfill, but unbounded daily JSON growth will eventually make `main` noisy.
Status: policy in place; compactor deferred.

Policy: Keep the last 90 days at daily granularity in `snapshots/`. Compact older dailies to `snapshots-monthly/YYYY-MM.json`, using the value at month-end (or latest available snapshot in that month). Archive snapshots older than one year to a separate `archive` branch. Do not run compaction in ordinary fix dispatches; use `agent_docs/retention.md`.
