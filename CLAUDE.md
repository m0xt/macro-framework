# Macro Framework — Claude Context

## What this is
Macro regime indicator system. The dashboard is a four-step walkthrough of a single
unified signal (MRMI) and its two underlying pillars.

Dashboard at `.cache/dashboard_v2.html` (built by `build_v2.py`).
The legacy `.cache/dashboard.html` (built by `build.py`) is still produced but no
longer the primary surface — `dashboard_v2.html` is what we iterate on.

## Conceptual model

**MRMI — Milk Road Macro Index.** The headline regime signal (LONG vs CASH).
A single composite, > 0 means stay long, < 0 means move to cash. Built from:

```
MRMI = MMI + macro_buffer − threshold
macro_buffer = buffer_size × (1 − stress_intensity)   # buffer_size=1.0
stress_intensity = min(1, max(0, −Real_Economy) × max(0, Inflation_Direction))
threshold = 0.5  (baked into the MRMI value so > 0 = LONG)
```

The buffer keeps us invested by default; only when MMI turns red **and** macro
stress builds does MRMI cross below zero and trigger CASH.

**MMI — Market Momentum Index.** The fast, market-derived half of MRMI.
Equal-weighted (1/1/1) average of three market signals: GII, Breadth,
FinCon. Equal weights selected by drawdown-optimization grid search —
they deliver stronger Calmar ratios than the prior alpha-weighted
(37/35/28) scheme and survive OOS validation better.

**Macro Stress.** The slow, economy-derived half. A 0–1 score capturing how
deep we are inside the stagflation pocket. Multiplicative AND of weak Real
Economy (PCE, Sahm, Real Income, GDPNow) and rising Inflation Direction
(Δ Core CPI YoY over 6m). Stress fires (`stress_on`) when intensity > 0.5.

**No "Macro Seasons" / no "Spring/Summer/Fall/Winter".** That model was
retired — we now use the AND-gate Macro Stress signal.
**No "MRCI"** — the growth axis is just called "growth".

## Dashboard structure (build_v2.py → dashboard_v2.html)

The page is a 4-step walkthrough wrapped by a hero with the headline number:

- **Hero** — big MRMI value + LONG/CASH state, scale bar, "What's behind it"
  pillars panel (MMI state + Macro Stress on/off), and the AI-generated
  weekly top brief ("This week's read").
- **Step 1 · How the index has evolved** — MRMI history chart (white line, green
  shading for LONG regimes, red for CASH). Range tabs: 1Y / 2Y / 5Y / ALL. SPX
  toggle on by default; MMI / Russell / BTC togglable in the legend.
- **Step 2 · Market pillar (MMI)** — single-line MMI chart over time, weekly
  market pillar brief, then drivers scorecard (GII / Breadth / FinCon)
  expanded by default; click an individual row to expand its chart.
- **Step 3 · Economy pillar (Macro Stress)** — dual-line chart: Real Economy
  Score (white solid, left axis) + Inflation Direction Δ6m (amber dashed,
  right axis). Weekly economy pillar brief, then real-economy drivers
  scorecard (PCE / Sahm / Real Income / GDPNow).
- **Step 4 · Reference Library** — supplementary indicators not in the formal
  signal. Click rows to expand charts.

Section headers use circled step numbers + a pillar chip (gray for market,
amber for economy).

## AI briefs (3-tier, weekly Tuesday cadence)

Three briefs in a tiered hierarchy, generated by `generate_commentary.py`
which subprocess-calls the `claude` CLI (uses Claude Code subscription —
no API key needed, no Anthropic SDK):

1. **Market pillar brief** → `briefs/YYYY-MM-DD/market.md`
2. **Economy pillar brief** → `briefs/YYYY-MM-DD/economy.md`
3. **Top brief** (synthesis) → `briefs/YYYY-MM-DD/top.md`

Pillar briefs are generated FIRST. The top brief reads them in as context
and synthesizes — it builds on them rather than re-deriving.

Cadence: **lazy weekly Tuesday**. A brief is stale if its dated archive is
older than the most recent Tuesday on or before today. Designed for the
Tuesday macro-research meeting — running `build_v2.py` any time from Tue
onward picks up that week's refresh.

Briefs are git-tracked under `briefs/` (the dir is NOT in `.gitignore`)
so past weeks are preserved forever for review.

Force-refresh: `.venv/bin/python generate_commentary.py --force`.

## Key commands
- `.venv/bin/python build_v2.py` — rebuild the v2 dashboard (auto-triggers
  brief generation if any are stale)
- `.venv/bin/python build.py --open` — rebuild legacy dashboard
- `.venv/bin/python build.py --no-cache --open` — force fresh data
- `.venv/bin/python generate_commentary.py --force` — force-regenerate all
  three briefs regardless of cadence
- `.venv/bin/python optimize.py composite` — re-optimize MMI weights
- `.venv/bin/python optimize.py gii` — re-optimize an individual indicator

## Architecture
- `build.py` — fetches data, computes all indicators, writes the legacy
  dashboard and a daily snapshot to `.cache/snapshots/YYYY-MM-DD.json`.
- `build_v2.py` — reads the latest snapshot + raw data, renders
  `dashboard_v2.html`. Triggers `generate_all_briefs()`.
- `generate_commentary.py` — three-tier brief generation via `claude` CLI.
- `optimize.py` — grid-search backtester for indicators + composite.
- Data: Yahoo Finance + FRED CSV (no API key for data either).
- Dashboard: self-contained HTML with Chart.js, all data embedded as JSON.

## Indicators (MMI components — equal weights, ⅓ each)
- **GII** — fast_roc=21, slow_roc=126, z_len=504, fast-only mode.
  10 components: HYG, HY spread (inv), XLY/XLP, XLI/XLU, SPHB/SPLV, copper,
  VIX (inv), 10Y-2Y curve, WEI, BDRY.
- **Breadth** — lookback=63, 7 cyclical ETFs (SMH, IWM, IYT, IBB,
  XHB, KBE, XRT), SLX dropped.
- **FinCon** — lookback=252, VIX + MOVE + HY (no IG).

Production MRMI OOS alpha (last 30%, ~4.2 yr): SPX +9.6%, IWM +12.9%,
BTC +2.1%. SPX max drawdown −18.9% → −4.8%. Active ~85% of OOS window.
MMI standalone (no buffer) posts higher OOS alpha — that's the cost of
the buffer keeping us invested through false alarms in a quiet period.
The buffer pays off in stress events (e.g. 2018, 2020, 2022 walk-forward).
Re-run `.venv/bin/python backtest_production.py` to refresh these.

## Macro Stress inputs
- **Real Economy Score** — equal-weighted z-score (3y window) of: Real PCE
  YoY, Sahm Rule (inverted), Real Personal Income YoY, Atlanta Fed GDPNow.
- **Inflation Direction** — Δ Core CPI YoY over the last 6 months, in pp.
- **Stress fires** when both turn against the market simultaneously.

## What's been deprecated
- **Macro Seasons** (Spring/Summer/Fall/Winter quadrants) — retired. The
  current model uses the AND-gate Macro Stress signal instead.
- **MRCI** name — gone. The growth axis is just "growth".
- **Static banner_story** — replaced by AI top brief in the hero.

## Dependencies
Python `.venv`: yfinance, pandas, numpy, requests. No `anthropic` SDK
needed — briefs go through the `claude` CLI.
