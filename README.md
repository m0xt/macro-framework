# Macro Framework

A macro regime indicator system that signals when to be invested (LONG) vs in cash (CASH), built around a single unified composite — **MRMI (Milk Road Macro Index)** — that combines a fast market-derived pillar with a slow economy-derived pillar. Built for a portfolio manager's weekly research cadence: one number tells you what to do, two pillar charts tell you why, and three AI-generated briefs explain the read.

## Setup

```bash
git clone https://github.com/your-org/macro-framework
cd macro-framework
brew install uv  # if uv is not already installed
uv sync --extra dev

# Fallback during the transition if uv is temporarily unavailable:
# python3 -m venv .venv && .venv/bin/pip install -r requirements.txt
```

## Quick Start

```bash
# Build the dashboard (uses cached data if < 12h old, auto-refreshes briefs if stale)
.venv/bin/python build.py
open .cache/dashboard.html

# Force fresh data + rebuild
.venv/bin/python build.py --no-cache
```

The build pipeline is now one stage: `build.py` fetches data, computes indicators, writes the daily snapshot, and renders the dashboard.

## AI Briefs

The dashboard includes three AI-generated briefs in a tiered hierarchy, regenerated weekly on Tuesday cadence (designed for Tuesday macro-research meetings):

1. **Market pillar brief** — what the markets are signaling this week
2. **Economy pillar brief** — what the economy is signaling this week
3. **Top brief** — synthesis of the two pillars, displayed in the hero

Briefs are generated by subprocess-calling the `claude` CLI, so they use your Claude Code subscription rather than an Anthropic API key. No `ANTHROPIC_API_KEY` is required and the `anthropic` Python SDK is not a dependency.

```bash
# Force-regenerate all three briefs (otherwise lazy weekly cadence)
.venv/bin/python weekly_briefs.py --force
```

Past briefs are git-tracked under `briefs/YYYY-MM-DD/{top,market,economy}.md` and preserved forever for review.

## Dashboard Layout

The dashboard is a four-step walkthrough wrapped by a hero with the headline number.

### Hero — the at-a-glance read

The big MRMI value (e.g. `+1.14`), LONG/CASH state, a scale bar showing where the value sits between −3 and +5, a "What's behind it" panel with the two pillar states (MMI green/red and Macro Stress on/off), and the AI-generated weekly top brief.

### Step 1 — How the index has evolved

MRMI history chart. White line = MRMI value over time; green shading = LONG regime; red = CASH. Range tabs (1Y / 2Y / 5Y / ALL). Toggle SPX / Russell / BTC overlays in the legend to see how each asset moved during each regime. Backtest stats are in a click-to-expand panel below the chart.

### Step 2 — What the markets are signaling (Market pillar)

**MMI — Market Momentum Index.** The fast, market-derived half of MRMI: built entirely from price and volatility data — credit spreads, cyclical sector breadth, and financial-conditions volatility. Markets react quickly so MMI catches turning points early; the trade-off is they can also flash false alarms, which is why MMI alone never triggers a CASH call.

| Driver | What it measures | Speed | Weight |
|--------|------------------|-------|--------|
| GII (Growth Impulses Index) | 10-component economic momentum | Fast (21d ROC) | ⅓ |
| Sector Breadth | 7 cyclical sector ETFs | Medium (63d lookback) | ⅓ |
| Financial Conditions | VIX + MOVE + HY spread | Slow (252d lookback) | ⅓ |

Equal weights (1/1/1) — switched from prior alpha-weighting (37/35/28) per the drawdown-optimization grid search; equal weights deliver stronger Calmar ratios across SPX/IWM/BTC and survive OOS validation better.

The drivers scorecard is open by default; click any row to expand its individual chart.

### Step 3 — What the economy is signaling (Economy pillar)

**Macro Stress.** The slow, economy-derived half: built entirely from real-economy data and inflation trajectory. The chart shows the two underlying axes — Real Economy Score (white solid, left axis) and Inflation Direction Δ6m (amber dashed, right axis). Stress fires only when growth is weak **and** inflation is rising — that AND condition filters out the false alarms the market pillar would otherwise produce on its own.

The four real-economy drivers feeding the score: PCE (consumer growth), Sahm Rule (forward-looking labor stress), Real Personal Income, Atlanta Fed GDPNow.

### Step 4 — Reference Library

Supplementary indicators not in the formal signal — kept for context, divergence-spotting, and as candidates for promotion into the pillars.

## How MRMI is computed

```
Stress_intensity = min(1, max(0, −Real_Economy) × max(0, Inflation_Direction))
Macro_buffer     = buffer_size × (1 − Stress_intensity)         # buffer_size = 1.0
MRMI             = MMI + Macro_buffer − threshold                # threshold = 0.5

MRMI > 0  → LONG  (stay invested)
MRMI < 0  → CASH  (step aside)
```

The buffer keeps the framework invested by default; only when MMI turns red **and** macro stress builds does MRMI cross below zero. Either pillar alone is not enough.

## Backtest Results (out-of-sample · last 30% of data, ~4.2 years)

Production MRMI signal (equal-weighted MMI + macro buffer with `buffer_size=1.0`, `threshold=0.5`):

| Asset | Buy & Hold | Signal-Only | Alpha | MaxDD reduction |
|-------|-----------|-------------|-------|-----------------|
| S&P 500 | +13.4%/yr | +23.0%/yr | **+9.6%/yr** | −18.9% → −4.8% |
| Russell 2000 | +12.2%/yr | +25.2%/yr | **+12.9%/yr** | −27.5% → −11.1% |
| Bitcoin | +26.2%/yr | +28.3%/yr | **+2.1%/yr** | −49.7% → −46.6% |

Active ~85% of OOS window (the buffer is designed to keep us invested through false alarms — see Walk-Forward for years where the buffer earned its keep). ~7 flips/year, ~35-day average regime. Full-sample SPX alpha is **+11.2%/yr** with max drawdown −10.1% (vs −33.9% buy & hold).

10-year backtest (2016–2026), no leverage, vs buy-and-hold. Walk-forward shows positive SPX and IWM alpha in **10 out of 10 years** (2017–2026); BTC averages roughly flat but the median year is positive.

### Honest caveat: MMI standalone vs MRMI

In the current OOS window, **MMI on its own posts higher alpha than MRMI** (SPX +12.6% vs +9.6%; IWM +20.3% vs +12.9%; BTC +13.4% vs +2.1%). That's by design, not a defect: the macro buffer is built to keep us invested through false alarms, and the OOS window happens to contain no real stagflation event — so the buffer pays a tax for protection it wasn't asked to provide. The buffer's value shows up in stress periods (2018, 2020, 2022 in walk-forward) where MRMI sidesteps the worst of every major SPX drawdown of the decade. If you operate the framework only during calm bull markets, you'd be better off with bare MMI; if you operate it across a full cycle, the buffer earns its keep when stress fires.

## Optimized Parameters

Grid-search optimized with 70/30 in-sample/out-of-sample split (2016–2023 / 2023–2026).

### GII
- `fast_roc=21`, `slow_roc=126`, `z_len=504`, no EMA
- Mode: fast composite only

### Financial Conditions
- `lookback=252`, components: VIX + MOVE + HY spread (IG dropped)

### Sector Breadth
- `lookback=63`, 7 ETFs (SMH, IWM, IYT, IBB, XHB, KBE, XRT), SLX dropped

### Macro Stress
- Real Economy Score: equal-weighted z-score (3y window) of Real PCE YoY, Sahm Rule (inverted), Real Personal Income YoY, Atlanta Fed GDPNow
- Inflation Direction: Δ Core CPI YoY over the last 6 months, in pp
- Stress fires when intensity > 0.5; threshold + buffer_size optimized for Calmar ratio

## Supabase Sync (optional)

Mirror computed indicators to a Supabase project for reuse in other apps.

### One-time setup

1. Apply `supabase_schema.sql` in your Supabase project's SQL Editor.
2. Copy `.env.example` to `.env` and fill in `SUPABASE_URL` + `SUPABASE_SERVICE_KEY`.
3. Backfill history: `.venv/bin/python sync_to_supabase.py backfill`

### Daily sync

After each dashboard build:
```
.venv/bin/python build.py && .venv/bin/python sync_to_supabase.py latest
```

### Downstream usage (any other project)

```python
from supabase import create_client
client = create_client(SUPABASE_URL, SUPABASE_ANON_KEY)
latest = (client.table('macro_snapshots')
                 .select('date,mrmi,mrmi_state,mmi,real_economy')
                 .order('date', desc=True).limit(1).execute()).data[0]
```

The anon key is safe to embed in browser bundles — RLS permits read only.
