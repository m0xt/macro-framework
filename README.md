# Macro Framework

Macro allocation-posture dashboard for deciding LONG (100% exposure), CAUTION (75% exposure), or CASH (0% exposure). It ingests Yahoo/FRED/DBnomics data, computes MRMI from market momentum plus macro stress, writes daily snapshots, renders a self-contained dashboard, syncs optional Supabase hot fields, and refreshes plain-English Claude weekly briefs on the first successful build on/after Tuesday.

## What ships today

- Hero with the current MRMI value, LONG/CAUTION/CASH posture, exposure weight, subtle threshold scale bar, pillar states, and the top weekly brief.
- MRMI history chart with LONG / CAUTION / CASH regime shading, SPX/Russell/BTC/MMI overlays, and a strengthened amber CAUTION band.
- Market pillar with MMI history, weekly market brief, and expandable Growth Impulses / Sector Breadth / Financial Conditions driver rows.
- Input drill-downs for all three MMI drivers: tooltip-backed input explanations, Input / Group / Current z / 7d zΔ / 30d zΔ tables sorted by |current z|, deterministic mini-briefs, and raw input history charts.
- Economy pillar with 0–10 Macro Stress, Real Economy Score + Inflation Direction inputs, weekly economy brief, and Real Economy driver rows.
- Reference Library for context indicators, including official CPI headline/core, PPI all commodities, ISM Manufacturing PMI via DBnomics, GDPNow, M2, CFNAI, industrial production, housing, permits, and claims.

## Quick start

```bash
brew install uv  # if needed
uv sync --extra dev
uv run pytest
uv run ruff check .
uv run python -m macro_framework.build
open outputs/dashboard.html
```

Force fresh data:

```bash
uv run python -m macro_framework.build --no-cache
```

Production refresh path:

```bash
scripts/refresh.sh                  # data/dashboard
scripts/refresh.sh --briefs-only   # force briefs, then rerender/dashboard sync
```

## Supabase sync

```bash
cp .env.example .env  # fill SUPABASE_URL + SUPABASE_SERVICE_KEY
uv run python -m macro_framework.sync_to_supabase doctor
uv run python -m macro_framework.sync_to_supabase latest
```

Remote schema must match the `migrations/` files (highest `000N_*.sql`) and `EXPECTED_SCHEMA_VERSION` in `src/macro_framework/sync_to_supabase.py`.

## Weekly briefs

Briefs live under `briefs/YYYY-MM-DD/{market,economy,top}.md` and are generated by the `claude` CLI. Manual builds use lazy Tuesday semantics unless forced; production forces briefs every weekday at 4:05pm ET, immediately after the 4:00pm ET data/dashboard refresh.

```bash
uv run python -m macro_framework.weekly_briefs --force
```

The dashboard renders the top brief in the hero and the market/economy briefs below their pillar charts. Current prompts favor plain-English explanations for non-macro colleagues over dense market jargon.

## Parameter locks

- Sector Breadth: `lookback=90` over 90 days.
- MRMI: `buffer_size=0.5`, `threshold=0.75`, `stress_p99=10.0083`, unified stress clipped to `[0, 1]`, 0–10 `stress_score` buckets at 3/5/7.
- Posture thresholds: MRMI < `-0.50` = CASH / 0%, `-0.50` through `+0.25` = CAUTION / 75%, MRMI > `+0.25` = LONG / 100%.
- Release lags: PCE/RPI 60d, unemployment 35d, Core CPI 45d, GDPNow 0d.

These are locked by `tests/test_smoke.py`; do not change docs or math independently.
